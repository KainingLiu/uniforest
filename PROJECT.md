## 1. 机器人是什么

我们这个机器人有**三个运动系统**，各自独立工作：

| 系统                      | 干什么的                         | 用什么驱动                  |
| ------------------------- | -------------------------------- | --------------------------- |
| **底盘**（4个麦轮）       | 让机器人在平面上前进/后退/转向   | 4个 M3508 电机 + C620 电调  |
| **机械臂**（2个步进）     | 水平（前后）和垂直（上下）移动   | 2个步进电机 + TB6600 驱动器 |
| **末端执行器**（4个舵机） | 夹爪开合、前臂升降、两个舱门开关 | 4个 PWM 舵机                |

**控制芯片**：STM32F427IIHx（ARM Cortex-M4，180MHz，2MB Flash，256KB RAM）

**程序架构**：裸机（Bare Metal），没有操作系统（RTOS），只有一个超级大循环（Super Loop）。

## 4. 程序是怎么跑起来的

### 4.1 启动流程

```
上电/复位
  │
  ▼
HAL_Init()                 ← 初始化 HAL 库（SysTick、Flash 等）
  │
  ▼
SystemClock_Config()       ← 配置时钟：外部12MHz晶振 → PLL倍频 → 180MHz
  │                          HSE=12MHz, PLLM=6, PLLN=180, PLLP=2
  │                          公式：SYSCLK = 12/6 × 180/2 = 180MHz
  ▼
MX_GPIO_Init()             ← CubeMX 自动生成的 GPIO 初始化
MX_CAN1_Init()             ← CAN1 外设初始化（1Mbps）
MX_CAN2_Init()             ← CAN2 外设初始化（暂时没用）
  │
  ▼
Motor3508_Init()           ← 启动 CAN1，配置滤波器，给C620上电(24V)，初始化PID
  │                          发送零力矩 → 电机锁死
  ▼
Servo_Init()               ← 配置 TIM2(CH2/3/4) + TIM5(CH1) 输出 50Hz PWM
  │                          4个舵机先输出中位脉宽(750 = 90°)
  ▼
Stepper_Init()             ← 配置步进 GPIO（PUL/DIR/ENA），初始状态=LOW
  │
  ▼
配置 A板 LED               ← PE11(红) PF14(绿) 输出推挽，初始关（SET=灭）
  │
  ▼
开机动画：红绿灯同闪 3 次   ← 每次150ms亮，150ms灭
  │
  ▼
HAL_Delay(1000)            ← 等1秒让系统稳定
  │
  ▼
Actions_ServoHome()        ← 4个舵机归位：夹爪81° 前臂90° 舱门A63° 舱门B117°
  │
  ▼
绿灯快闪一次              ← 表示"我准备好了"
  │
  ▼
进入主循环 →→→→→→→→→→→→→→→→
```

### 4.2 主循环在干什么（每 10ms 一次）

```
while (1):
  ┌─────────────────────────────────────────┐
  │ 1. Motor3508_RxPoll()                   │
  │    从 CAN1 接收队列里把所有反馈帧取出来  │
  │    更新每个电机的角度、转速、力矩、温度  │
  │    追踪累计位置（处理编码器过零）        │
  ├─────────────────────────────────────────┤
  │ 2. 检测按键                             │
  │    → 检测到上升沿（按下）→ 去抖 30ms    │
  │    → 确认有效：绿灯快闪，click_count++   │
  │    → 等待按键释放                       │
  │    → 如果距离上一次按下超过 1500ms：     │
  │        确认点击次数，执行对应动作        │
  │          1击: Motor3508_ChassisDemo()   │
  │          2击: Actions_Grap()           │
  │          3击: Actions_Build()          │
  ├─────────────────────────────────────────┤
  │ 3. HAL_Delay(10)  ← 延时10ms           │
  │    (约 100Hz 主循环频率)                │
  └─────────────────────────────────────────┘
```

**为什么是 100Hz？** 因为主循环每 10ms 跑一次：
- CAN 反馈每 1ms 就有一帧（电机以 1kHz 回报），`RxPoll()` 每次会清空整个 FIFO
- 10ms 对于按键检测绰绰有余（人手按一下起码 100ms）
- 舵机和步进动作是**阻塞式**的（执行期间主循环暂停），所以循环快慢不影响动作精度

### 4.3 什么是"裸机超级循环"？

没有 RTOS = 没有多线程 = 没有任务调度。程序就是一个 `while(1)` 不停地转。

**优点**：
- 代码简单直观，新手能看懂
- 没有任务切换开销
- 没有并发 bug（共享变量竞争等）

**缺点**：
- 一个动作阻塞期间，按键无法响应
- 不能同时做两件事（比如不能一边走底盘一边操作舵机）
- 所有延时都是 `HAL_Delay()`（忙等），CPU 空转

## 5. 底盘电机 — M3508

### 5.1 M3508 是什么

DJI RoboMaster 专用的三相无刷直流减速电机，自带编码器和温度传感器。配 C620 电调（电子调速器），电调内部自己做了电流闭环（FOC 控制），我们只需要通过 CAN 总线告诉它"输出多少力矩"就可以了。

**关键参数**：

| 参数         | 值                 | 意义                               |
| ------------ | ------------------ | ---------------------------------- |
| 编码器分辨率 | 8192 counts/rev    | 电机转一圈编码器输出 0~8191        |
| 减速比       | 19:1               | 电机转19圈 = 输出轴转1圈           |
| 轮子直径     | 152mm              | 麦轮外径                           |
| 力矩范围     | -16000 ~ +16000    | CAN 指令的原始数值，正负对应正反转 |
| 最大转速     | ~469 rpm（输出轴） | 减速后                             |

### 5.2 4个电机怎么分布的

```
         机器人俯视图
    ┌──────────────────┐
    │                  │
    │   TL(1)  TR(0)   │  ← 电机编号（CAN反馈ID）
    │   左上    右上    │     TR=Top Right, TL=Top Left
    │                  │     BL=Bottom Left, BR=Bottom Right
    │   BL(2)  BR(3)   │
    │   左下    右下    │
    │                  │
    └──────────────────┘
```

| 编号   | 位置 | CAN 反馈 ID | 前进时转哪边  |
| ------ | ---- | ----------- | ------------- |
| 0 (TR) | 右上 | 0x201       | **反转**（-） |
| 1 (TL) | 左上 | 0x202       | **正转**（+） |
| 2 (BL) | 左下 | 0x203       | **正转**（+） |
| 3 (BR) | 右下 | 0x204       | **反转**（-） |
CAN ID为对应编号+1

**为什么对角反过来？** 因为麦轮的安装方向：左右两侧的麦轮小辊子方向是相反的，所以 TR 和 BR 往同一个方向转时，对地面的推力才一致。

> 改成后退：把上面的符号全部反过来就行。

### 5.3 CAN 通信协议

CAN（Controller Area Network）是工业界常用的现场总线，我们这叫 CAN1，速率 1Mbps。

**发送（A板 → 电机）**：

| 字段     | 值                                                 |
| -------- | -------------------------------------------------- |
| CAN ID   | `0x200`（标准帧）                                  |
| 数据长度 | 8 字节                                             |
| 字节布局 | `[M1高, M1低, M2高, M2低, M3高, M3低, M4高, M4低]` |
| 编码方式 | 大端序（Big Endian），每2字节一个 int16            |

> 例：给4个电机发 [1000, -500, 2000, 0]，数据就是 `03 E8 FE 0C 07 D0 00 00`

**接收（电机 → A板）**：

每个电机用自己的 CAN ID 回报数据：

| 电机       | CAN ID | 数据含义                                                            |
| ---------- | ------ | ------------------------------------------------------------------- |
| 电机1 (TR) | 0x201  |                                                                     |
| 电机2 (TL) | 0x202  | 8字节：[角度高, 角度低, 转速高, 转速低, 力矩高, 力矩低, 温度, 保留] |
| 电机3 (BL) | 0x203  |                                                                     |
| 电机4 (BR) | 0x204  |                                                                     |

- **角度**：uint16，0~8191，表示转子机械角度（一整圈 = 8192）
- **转速**：int16，单位 RPM（转/分钟），正值 = 正转
- **力矩**：int16，-16000 ~ 16000，实际输出的力矩电流
- **温度**：uint8，单位℃，超过 80℃ 要警惕

### 5.4 PID 控制（重要！）

#### 5.4.1 为啥需要 PID

你不能直接说"往前走 20cm"然后给电机一个恒定力矩——那样会因为地面摩擦力不同、电池电压变化而导致走不准。

PID 是一个**闭环控制算法**：你想要某个值（设定点），实际值是另一个数，PID 帮你算出"需要加多大的力来消除误差"。

#### 5.4.2 三个字母啥意思

```
P（比例项） = Kp × 误差
  → 误差越大，出力越大。就像弹簧，拉得越远弹力越大。

I（积分项） = Ki × 误差累计
  → 把过去的所有误差加起来。专门对付"稳态误差"
  → 比如轮子摩擦大，P不够力，I就慢慢帮你把力叠上去。

D（微分项） = Kd × 误差变化率
  → 预测误差趋势。快到目标了提前减速，防止冲过头。
  → 我们目前没用（Kd = 0）
```

**积分限幅（Anti-Windup）**：如果电机卡住了，误差一直很大，积分会变得巨大（"饱和"），导致解锁后突然猛冲。所以我们要限制积分的最大值 `integral_limit`。

#### 5.4.3 我们的 PID 架构

```
位置设定 → [位置PID] → 速度设定 → [速度PID] → 电流设定 → [C620自己] → 电机
   ↑                                                        ↓
   └────── 编码器反馈(累计位置) ←─── 速度反馈 ←─── 力矩反馈 ←───┘
```

这是**级联 PID**（Cascaded PID），三层套娃：
1. **位置环**（最外层）：我说"走 20cm"，它算出"当前应该以多少 RPM 转"
2. **速度环**（中间层）：接收目标 RPM，算出"应该输出多少力矩电流"
3. **电流环**（最内层）：C620 电调硬件自己做的，我们不用管

**当前代码只用速度环**（ChassisDemo 里面），位置环的代码写好了但没用上。

#### 5.4.4 PID 参数（可以调！）

| 环     | Kp      | Ki       | Kd  | 积分限幅 | 输出限幅 |
| ------ | ------- | -------- | --- | -------- | -------- |
| 位置环 | 0.02    | 0.0005   | 0   | ±300 RPM | ±600 RPM |
| 速度环 | **7.0** | **0.03** | 0   | ±1500    | ±5000    |
| 电流环 | 0.5     | 0        | 0   | ±1000    | ±5000    |

> 当前只用了速度环，所以只有 `Kp=7.0` 和 `Ki=0.03` 在生效。
> 调参口诀：先把 Ki 设为 0，调 Kp 到不抖；再加 Ki 到能稳住不动。

### 5.5 ChassisDemo 是怎么走的（按键1击）

```
前进阶段：梯形速度曲线

  速度(RPM)
    │
  800 ┤          ┌──────────────────┐         ← 匀速（Cruise）
    │          ╱                    ╲
    │        ╱                        ╲
    0 ┤─────┘                          └──────→ 时间
    │←300ms→│←───── 800ms ──────→│←300ms→│←200ms→│
      加速       匀速前进            减速     保持
```

1. **加速**（300ms）：速度从 0 线性拉到 800 RPM
2. **匀速**（800ms）：保持 800 RPM → 机器人前进约 20cm
3. **减速**（300ms）：速度从 800 线性降到 0
4. **保持**（200ms）：发送零力矩刹车
5. **打开舱门** → 暂停1秒
6. **后退**（同样的梯形速度曲线，反向）
7. **关闭舱门**

**底盘距离计算**：

```
轮子转一圈的周长 = π × 152mm ≈ 477.5mm
电机输出轴转一圈 = 编码器转 1/19 圈 = 8192/19 ≈ 431 个编码计数
每厘米需要 = 编码器每圈 × 减速比 / 周长 × 1cm
           = 8192 × 19 / 477.5 × 10
           ≈ 3260 counts/cm
```

所以走 20cm ≈ 目标 65200 counts。

**麦轮前进的方向矩阵**（800 RPM 基准）：

```c
正向(前进)：[TR=-800,  TL=+800,  BL=+800,  BR=-800]
反向(后退)：[TR=+800,  TL=-800,  BL=-800,  BR=+800]
```

## 6. 舵机 — Servo

### 6.1 舵机是怎么控制的

舵机通过 **PWM 脉冲宽度** 来控制角度：

```
标准舵机信号（50Hz = 20ms 周期）：
  
  0.5ms ────→ 0°
  1.0ms ────→ 45°
  1.5ms ────→ 90° （中位）
  2.0ms ────→ 135°
  2.5ms ────→ 180°

  0ms    1ms    2ms    3ms     ...     20ms
  ├──────┼──────┼──────┼──────┤         │
  │ HIGH │ LOW                │  下一个  │
  ├──────┼────────────────────┤  周期    │
       ↑
   脉宽（决定角度）
```

### 6.2 4个舵机是什么

| ID  | 名称     | 引脚 | 定时器/通道 | 功能       | 归位角度     |
| --- | -------- | ---- | ----------- | ---------- | ------------ |
| 0   | S1 夹爪  | PA1  | TIM2_CH2    | 抓/放物体  | 81°（张开）  |
| 1   | S2 前臂  | PA2  | TIM2_CH3    | 手臂抬/降  | 90°（放下）  |
| 2   | S3 舱门A | PA3  | TIM2_CH4    | 舱门A开/关 | 63°（关闭）  |
| 3   | S4 舱门B | PA0  | TIM5_CH1    | 舱门B开/关 | 117°（关闭） |

### 6.3 定时器配置细节

```
APB1 定时器时钟 = 90MHz（APB1=45MHz, ×2 because prescaler≠1）
目标 PWM 频率 = 50Hz

PSC（预分频器）= 179   → 90MHz / 180 = 500kHz (每tick = 2μs)
ARR（自动重装载）= 9999 → 500kHz / 10000 = 50Hz

脉宽换算：
  0.5ms / 2μs = 250 ticks  → 0°
  1.5ms / 2μs = 750 ticks  → 90°
  2.5ms / 2μs = 1250 ticks → 180°

角度 → 脉宽公式：
  pulse = 250 + angle × 1000 / 180
```

### 6.4 关键函数

| 函数                      | 作用                                       |
| ------------------------- | ------------------------------------------ |
| `Servo_Init()`            | 配置 GPIO AF + TIM2/TIM5 PWM 输出，初始90° |
| `Servo_SetAngle(id, deg)` | 设定某个舵机的角度（0~180，自动限幅）      |
| `Servo_HomeAll()`         | 全部归位                                   |
| `Servo_HomeSingle(id)`    | 单个归位                                   |

> **注意**：舵机是开环控制的！代码只管设角度，不管舵机有没有真的转到。所以 `HAL_Delay(500)` 是必须的——给舵机 500ms 时间完成动作。如果舵机还没转到你就发下一条指令，它会直接切过去。

## 7. 步进电机 — Stepper

### 7.1 步进电机是什么

步进电机是一步一步转的——给一个脉冲信号就转一个固定角度。我们用的是 TB6600 驱动器，接线方式为共阴极。

**关键参数**：

| 参数     | 值              | 意义                     |
| -------- | --------------- | ------------------------ |
| 步距角   | 1.8° (400步/圈) | 2细分微步后等效 400步/圈 |
| 丝杠导程 | 10mm            | 丝杠转一圈推10mm         |
| 步/厘米  | 400             | 所以你走1cm = 400步      |
| 启停频率 | ~500Hz          | 直接起步不会丢步         |
| 巡航频率 | ~5kHz           | 加速后才能达到           |

### 7.2 三根控制线

| 线         | 引脚（1#水平） | 引脚（2#垂直） | 作用                            |
| ---------- | -------------- | -------------- | ------------------------------- |
| PUL (脉冲) | PI0            | PH10           | 每个上升沿 = 走1步              |
| DIR (方向) | PH12           | PD15           | LOW=正向, HIGH=反向             |
| ENA (使能) | PH11           | PD14           | LOW=电机上电锁住, HIGH=电机自由 |

**脉冲时序**（每次一步）：

```
PUL: __/‾‾‾‾‾‾‾\_______   HIGH时间 = 半步周期
         ← delay_us →       LOW时间 = 半步周期
```

`delay_us` 决定了转速：
- 1000μs 半步周期 → 500Hz → 1.25cm/s（启动速度）
- 100μs 半步周期 → 5kHz → 12.5cm/s（巡航速度）

### 7.3 梯形加减速

直接以高速启动，步进电机会**丢步**（脉冲发了但电机没转）。所以需要梯形加速：

```
速度(频率)
  │
 5kHz ┤          ┌──────────────┐         ← 匀速段（target_delay = 100μs）
  │          ╱                    ╲
  │        ╱                        ╲
500Hz ┤──┘                            └──→ 时间
  │←加速段→│←── 匀速段 ──→│←减速段→│
  │ accel_steps 步       │ accel_steps 步
```

- **加速段**：delay 从 1000μs 逐步降到 100μs（速度从 500Hz 升到 5kHz）
- **匀速段**：保持 100μs 半边周期不变
- **减速段**：delay 从 100μs 逐步升回 1000μs

如果总步数太少装不下完整梯形（`steps < 2×accel_steps`），代码会自动把加速段砍半。

### 7.4 双电机同步 — MoveOverlap 算法

这是整个代码里最巧妙的算法。有时候需要两个步进**同时动**但**各自速度不同**——比如水平走 23cm 的同时垂直走 5cm。

**核心思路**：用一个"快速时钟"统一两个电机的节奏。

```
假设电机A delay=200μs，电机B delay=300μs
快速时钟取 min(200,300) = 200μs（跟快的一起走）

在这 200μs 里：
  电机A 该走 1 步（刚好 200 = 200×1.0）
  电机B 该走 200/300 = 0.667 步（不够 1 步，先记账）

累计到下一次：0.667 + 0.667 = 1.334 ≥ 1.0 → 走 1 步，剩余 0.334
```

**固定点小数实现**（避免浮点运算影响速度）：

```c
#define SUB_SHIFT  10
#define SUB_ONE    (1 << 10)    // = 1024，表示"1.0步"

acc += (fast_delay << 10) / motor_delay;    // 累加本次"几分之一步"
if (acc >= 1024) { 发脉冲; acc -= 1024; }   // 攒够一整步就发
```

### 7.5 关键函数

| 函数                                               | 作用                                 |
| -------------------------------------------------- | ------------------------------------ |
| `Stepper_Init()`                                   | GPIO 初始化，PUL=高速, DIR/ENA=低速  |
| `Stepper_SetDir(id, dir)`                          | 设方向                               |
| `Stepper_Enable(id, ena)`                          | 使能/失能                            |
| `Stepper_MoveAccel(id, steps, ...)`                | 单电机梯形运动（阻塞）               |
| `Stepper_MoveOverlap(m1, s1, m2, s2, offset, ...)` | 双电机同步运动（阻塞）               |
| `Stepper_MoveOverlap2(...)`                        | 双电机同步，其中一个中途换向（阻塞） |
| `delay_us(us)`                                     | NOP 轮询微秒延时（针对 180MHz 校准） |
| `CM_TO_STEPS(cm)`                                  | 厘米 → 步数转换宏                    |

### 7.6 Stepper_MoveOverlap2 详解

用于复杂的机械臂动作：电机A一路往前走，电机B先正转10cm，然后停下来等，等A走到某个位置后B再反转5cm。

```
电机A（连续）  ████████████████████████████████████████
电机B（分相）  ██████(正转)▏......等待......▏████(反转)
                ←ph1→                ←ph2→
                           ↑ ph2_offset
                          A走到这里B开始反转
```

## 8. 动作序列 — Build 和 Grab

这两个是最上层的"剧本"，把舵机和步进的动作串联起来，形成完整的任务。

### 8.1 Grab（7步抓取）

```
  1. 舵机归位（夹爪打开、手臂放下、舱门关）
  2. 打开两个舱门
  3. 水平前伸22cm，同时垂直下降18cm  → 到达目标位置
  4. 夹爪合拢（抓住物体）
  5. 垂直上升18cm，同时水平后退22cm  → 拿起物体回来
  6. 夹爪打开（放下物体）
  7. 舵机归位、关舱门
```

水平/垂直都有平衡校验：
```
水平：+22 -22 = 0 ✓  （回到原位）
垂直：-18 +18 = 0 ✓
```

### 8.2 Build（24步搭建）

Build 是一个很长的搭积木动作，24步。核心模式是：

1. 前伸抓取 → 夹爪合拢
2. 手臂抬起（平滑减速避限位）
3. 运送到目标位置 → 夹爪张开
4. 退回原位

**手臂抬起的平滑减速**：因为舵机有机械限位（0°），直接用 `Servo_SetAngle(0)` 会撞上去。所以分6小步逐步逼近：

```c
Servo_SetAngle(ARM, 50);  HAL_Delay(80);   // 50°
Servo_SetAngle(ARM, 28);  HAL_Delay(90);   // 28°
Servo_SetAngle(ARM, 14);  HAL_Delay(90);   // 14°
Servo_SetAngle(ARM, 6);   HAL_Delay(90);   // 6°
Servo_SetAngle(ARM, 2);   HAL_Delay(100);  // 2°
Servo_SetAngle(ARM, 0);   HAL_Delay(50);   // 0° ← 刚好到限位
```

水平/垂直位移平衡校验（注释里写的）：
```
水平: +5+18-23+23-23+23-23 = 0  ✓（最终回到原位）
垂直: -1-18+10-2-3+5-12+21-4+4 = 0  ✓
```

> **两个函数都是阻塞式的**——执行期间主循环什么都不干，按键也失效。Build 可能要好几分钟才能跑完。

## 9. 单按键多击 — 操作方式

整个机器人只有**一个按键**（PB2），通过不同点击次数来区分指令。

### 9.1 按法

```
  开机 → 闪烁3次 → 绿灯快闪一次 → 就绪
    │
    按1下 → 绿灯闪1下确认 → 执行底盘Demo
    按2下 → 绿灯闪2下确认 → 执行Grab
    按3下 → 绿灯闪3下确认 → 执行Build
    按4下以上 → 忽略
```

### 9.2 判断逻辑

```
状态机：

每次检测到按键上升沿（0→1）：
  → 去抖30ms后再确认一次（防抖动）
  → 确认成功：click_count++
  → 等待按键释放（while KEY_PRESSED 死等）
  → 记录按下时间戳 last_click_tick

每次主循环检查：
  → 如果 click_count ≥ 1 且距今 ≥ 1500ms（多击窗口关闭）
      → 根据 click_count 执行动作
      → 重置计数器
```

**多击窗口** = 1500ms。意思是：你按完第一下后，要在 1.5 秒内按第二下才算双击；超过 1.5 秒就按当前次数执行。

### 9.3 信号确认

| 事件     | LED 反馈                       |
| -------- | ------------------------------ |
| 按键按下 | A板绿灯快闪一次（80ms）        |
| 动作确认 | A板绿灯闪烁N次（N=点击次数）   |
| 开机完成 | A板红绿灯同闪3次 + 绿灯快闪1次 |

## 12. 常见问题 & 调试技巧

### 12.1 电机不转？

1. 24V 电池接了吗？C620 电调灯亮吗？
2. CAN 线接对了吗？CAN_H 和 CAN_L 没接反吧？
3. 终端电阻加了吗？（CAN 总线两端各需要一个 120Ω）
4. 用示波器看一下 CAN 总线上有没有波形

### 12.2 电机抖/跑飞？

1. PID 参数可能太大了——降 Kp 试试（`M3508_SPEED_KP`）
2. 检查编码器反馈：用调试器看 `g_motor[i].feedback.speed_rpm` 是不是合理
3. CAN 丢帧：检查 `Motor3508_RxPoll()` 调用频率够不够

### 12.3 舵机不动？

1. TIM2/TIM5 时钟开了吗？（代码里 `__HAL_RCC_TIM2_CLK_ENABLE()`）
2. GPIO Alternate Function 配对了吗？PA1→AF1, PA0→AF2
3. 用示波器看 PA0~PA3 有没有 50Hz PWM 波形

### 12.4 步进丢步？

1. 启动速度太快了：`STEP_START_DELAY_US` 调大点（现在 1000μs = 500Hz）
2. 加速段太短了：`STEP_ACCEL_STEPS` 调大点（现在 400 步）
3. 机械负载太大：检查丝杠润滑、皮带张力

### 12.5 程序卡死了？

1. 接上 ST-Link，在 KEIL/CLion 里打断点
2. 用 `HAL_GetTick()` 看看是不是某个 `while` 循环出不来
3. 检查是不是在 Build() 执行期间没耐心（要跑好几分钟呢！）

### 12.6 调试时的好习惯

1. **LED 是最简单的调试工具**：在关键位置加 LED 闪烁
2. **用 ST-Link 看变量**：IDE 断点在 `Motor3508_RxPoll()` 里看 `g_motor[0]` 的值
3. **先单独测每个模块**：单独测电机、单独测舵机、单独测步进，确认都没问题了再跑动作序列
4. **改 PID 参数要小步改**：每次改 10%~20%，然后实测效果

## 13. 改代码指南

### 13.1 我想让机器人走更远

改 `motor3508.h` 里的：

```c
#define DEMO_DISTANCE_CM    20    // ← 改成你想要的厘米数
```

### 13.2 我想让机器人走慢点

改 `motor3508.h` 里的：

```c
#define DEMO_SPEED_RPM      800   // ← 降到 400 就是一半速度
```

### 13.3 我想调 PID

改 `motor3508.h` 里的 `M3508_SPEED_KP` 和 `M3508_SPEED_KI`。

调参口诀：
1. `Ki = 0`，慢慢加 `Kp` 直到电机有点抖（震荡），然后退回 70%
2. 慢慢加 `Ki`，直到能消除静差（走到目标位置后不漂移）

### 13.4 我想改 Grab/Build 的步进步数

改 `actions.c` 里对应的 `N * STEPS_PER_CM`（N 是厘米数）。注意！要保持水平/垂直的位移平衡，前进的总和应该等于后退的总和。

### 13.5 我想加新的按键动作

1. 在 `motor3508.c`（或新文件）写动作函数
2. 在 `main.c` 的 switch 里加新 case：

```c
case 4:  // 4击
    MyNewAction();
    break;
```

### 13.6 文件修改速查

| 想改什么       | 去哪里改                                                                          |
| -------------- | --------------------------------------------------------------------------------- |
| 底盘速度/距离  | `motor3508.h` → `DEMO_SPEED_RPM` / `DEMO_DISTANCE_CM`                             |
| PID 参数       | `motor3508.h` → `M3508_SPEED_KP` 等                                               |
| 舵机角度定义   | `actions.h` → `ANGLE_*` 宏                                                        |
| 舵机归位角度   | `servo.h` → `SERVO1_HOME` 等                                                      |
| 步进加减速     | `stepper.h` → `STEP_START_DELAY_US` / `STEP_TARGET_DELAY_US` / `STEP_ACCEL_STEPS` |
| 步进每厘米步数 | `stepper.h` → `STEPS_PER_CM`（改这个会影响所有 `N * STEPS_PER_CM`）               |
| 多击窗口时间   | `main.c` → `CLICK_WINDOW_MS`                                                      |
| 主循环频率     | `main.c` → `POLL_INTERVAL_MS`                                                     |
| Grab 动作      | `actions.c` → `Actions_Grap()`                                                    |
| Build 动作     | `actions.c` → `Actions_Build()`                                                   |
